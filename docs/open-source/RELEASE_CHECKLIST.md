# Release Checklist

Use this before publishing a release or tagging a public version.

## Legal and Brand

* AGPL license notice is present
* trademark policy is published
* commercial contact path is correct
* disclaimers are in place

## Product Quality

* migrations run successfully
* seeders run successfully
* tests pass for core flows
* key user journeys are documented

## Public Repo Hygiene

* README links are current
* contribution guide is current
* security reporting path is current
* roadmap reflects the intended public scope

## Commercial Boundaries

* premium features remain outside the public core
* hosted-service value is not fully duplicated in the repo
* branding assets are protected

## Release Rule

If any item above is not ready, do not publish the release yet.
