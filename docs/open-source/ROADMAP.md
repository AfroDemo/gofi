# Public Roadmap

## Current Open-Source Focus

The public roadmap should stay centered on the core product foundation that helps contributors and operators:

* tenant management
* package and voucher workflows
* session control
* transaction tracking
* revenue sharing foundations
* router integration scaffolding

## Short-Term Public Priorities

* harden payment callback handling
* improve voucher lifecycle clarity
* expand reporting for operators
* strengthen tenancy and authorization boundaries
* improve test coverage for business-critical flows

## What Stays Off the Public Roadmap

Keep the following as private or premium-led work unless you intentionally decide otherwise:

* advanced analytics
* AI insights
* white-label controls
* enterprise dashboards
* multi-country payment expansion

## Roadmap Rule

If a feature mainly protects revenue, reduces support overhead, or improves commercial leverage, treat it as a premium candidate rather than a public default.
